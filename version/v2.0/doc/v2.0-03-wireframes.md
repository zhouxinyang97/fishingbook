# Fishing Book v2.0 PRD - Wireframes（页面与交互草图）

> 说明：Fishing Book 目前为桌面应用单窗口形态；v2.0 书库功能建议仍保持单窗口，通过“页面状态切换”实现。以下为低保真示意，元素含默认/加载/空/错误状态。

## Page A：书库选择（Library Picker）

**默认态**
```
┌──────────────────────────────────────────┐
│ Fishing Book                              │
├──────────────────────────────────────────┤
│ 选择书库                                   │
│ [最近打开]                                 │
│  - 个人小说库   D:\Books\Novels   [打开]   │
│  - 工作资料库   D:\Docs\Specs     [打开]   │
│                                            │
│ [打开其他书库…]  (选择一个文件夹作为书库)  │
│                                            │
│ 可选：启动时自动打开上次书库  [✓]          │
└──────────────────────────────────────────┘
```

**空态**
- “最近打开”为空时，突出主 CTA：打开其他书库…；附一句解释“书库=一个本地文件夹，用来组织你的书本”。

**错误态**
- 书库路径不存在/无权限：展示原因 + “从最近列表移除/重新选择路径”。

## Page B：书架（Bookshelf / Library Home）

**默认态**
```
┌──────────────────────────────────────────┐
│ [当前书库：个人小说库 ▼]   [导入书本] [设置]│
├──────────────────────────────────────────┤
│ 搜索： [ 关键字… ]                         │
│                                            │
│ 继续阅读：  射雕英雄传   进度 42%  [继续]   │
│ ───────────────────────────────────────── │
│ 书本列表                                   │
│  射雕英雄传.txt   更新:2026-02-01  [打开]  │
│  逆风飞扬.txt     更新:2025-12-12  [打开]  │
│  郁达夫诗全集.txt  更新:2025-11-03  [打开]  │
│                                            │
│ 右键/更多：重命名显示名 | 移除 | 在资源管理器中显示 │
└──────────────────────────────────────────┘
```

**空态**
- 无书本：展示插画/文案 + 主 CTA “导入书本”；说明支持 `.txt/.log/.md`（与现有能力一致）。

**错误态（单书缺失）**
- 列表项显示 “文件缺失” 标记；点“打开”→进入修复弹窗：“重新定位文件 / 从书库移除”。

## Page C：阅读器（Reader）

> 复用现有阅读器视觉（透明窗口/日志伪装风格）。v2.0 重点是：打开来自书库的书本，并按书恢复/保存进度。

**默认态**
```
┌──────────────────────────────────────────┐
│ [← 书架]  射雕英雄传.txt        [置顶] [外观]│
├──────────────────────────────────────────┤
│  (正文区域：滚动)                           │
│  ...                                       │
│  自动保存进度提示（可选，短暂 Toast）        │
└──────────────────────────────────────────┘
```

**外观设置弹窗（全局设置）**
```
┌──────────────────────────────────────────┐
│ 外观设置（全局）                     [×]   │
├──────────────────────────────────────────┤
│ 透明度： [=====●======]  60%              │
│ 主题色：  ( ) VS Code Dark+               │
│          ( ) One Dark                     │
│          ( ) Light                        │
│                                          │
│ [恢复默认]                      [完成]    │
└──────────────────────────────────────────┘
```

**加载态**
- 读文件、解码与渲染期间：正文区展示轻量 Loading（不阻塞窗口拖动/置顶/透明度）。

**错误态**
- 解码失败/文件读取失败：正文区展示错误原因 + “返回书架 / 重新定位文件”。

**窗口拖拽缩放（与 v1.0 一致）**
- 用户可拖拽窗口四边/四角改变窗口尺寸。
- 缩放过程中正文暂停重渲染，鼠标抬起后恢复渲染，避免大文本场景卡顿。

## 无障碍与可用性（最低要求）
- 键盘可达：书架“搜索框、导入、打开、继续阅读、返回书架”均支持 Tab 焦点与 Enter 触发。
- 字体与对比：保持暗色对比度 ≥ 4.5:1（参考 WCAG 2.2 AA）。
- 小窗口适配：宽 < 360px 时，书架列表切为单列；隐藏次要元信息（更新时间）。
