# Fishing Book v2.0 PRD - User Stories（用户故事）

## 1. 核心用户旅程（端到端）
- 发现/启动：首次启动进入“书库选择”→创建/选择一个书库→进入书架→导入书本→打开阅读器。
- 习惯循环：下次启动自动进入上次书库→书架顶部展示“继续阅读”→点击即恢复到该书上次位置。
- 切换：阅读器返回书架→选另一书→各自恢复进度。

## 2. 用户故事清单（按优先级）

### US-01 书库选择与打开（Must）
作为用户，我希望能选择并打开一个书库，以便在不同书库间切换管理不同书本集合。

**验收标准（Gherkin）**
```gherkin
Feature: Library picker
  Scenario: Open a recent library
    Given I have opened a library before
    When I launch the app
    Then I can see the recent libraries list
    And I can open one library in 1 click

  Scenario: Open a new library folder
    Given I am on the library picker page
    When I choose a folder path as a library
    Then the library is added to recent libraries
    And the app opens the library bookshelf
```

### US-02 导入书本到书库（Must）
作为用户，我希望在书库中添加书本（选择一个或多个文件），以便形成可管理的书架列表。

```gherkin
Feature: Import books
  Scenario: Import multiple files
    Given I am on a library bookshelf
    When I select 3 files to import
    Then 3 books appear in the list
    And duplicated files are not imported twice
```

### US-03 书本管理（Must）
作为用户，我希望在书架中管理书本（删除、修改显示名、搜索），以便快速找到要读的书。

```gherkin
Feature: Manage books
  Scenario: Rename display title
    Given a book exists in the bookshelf
    When I rename its display title
    Then the new title shows in the list
    And the underlying file path does not change

  Scenario: Remove a book
    Given a book exists in the bookshelf
    When I remove it
    Then it disappears from the list
    And its reading progress is removed or archived
```

### US-04 按书保存与恢复上次阅读位置（Must）
作为用户，我希望每本书能单独保存上次阅读位置，以便下次打开同一本书时自动续读。

```gherkin
Feature: Per-book reading progress
  Scenario: Save progress while scrolling
    Given I am reading a book
    When I scroll and stop for a short time
    Then the app saves the progress for this book

  Scenario: Restore progress when opening
    Given I have previously read this book to 40%
    When I open this book again
    Then the content scrolls to around 40%
```

### US-05 最近阅读/继续阅读入口（Should）
作为用户，我希望在书架页看到“继续阅读”，以便更快回到上次阅读的书。

```gherkin
Feature: Continue reading
  Scenario: Continue reading from bookshelf
    Given I have at least one book with saved progress
    When I open the bookshelf
    Then I can see a Continue Reading entry
    And clicking it opens the last-read book and restores progress
```

### US-06 文件丢失/移动的修复引导（Should）
作为用户，当书本文件被移动或删除时，我希望应用提示并允许我重新定位文件，以免书架失效。

```gherkin
Feature: Missing file recovery
  Scenario: Book file missing
    Given a book points to a non-existent file path
    When I click to open the book
    Then the app shows a missing-file error state
    And I can choose a new file path to relink
```

## 3. 主要情绪曲线与掉点
- 高点：一键打开书库、一键继续阅读、切换书本仍能精准续读。
- 掉点风险：首次创建书库概念不理解；导入重复/失败无反馈；文件丢失后无法修复导致信任下降。
